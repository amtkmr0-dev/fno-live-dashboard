# Hardening pass — 2026-05-26

A backup of the project as it stood before this change is at:
`~/Desktop/fno-live-dashboard_backup_2026-05-26_06-14-58/`

## Security fixes

1. **ws_server now binds to `127.0.0.1` by default.**
   Override with `WS_BIND_HOST` only if a separate firewall is in place.

2. **Signed identity between auth_proxy and ws_server (`internal_auth.py`).**
   Previously ws_server trusted plaintext `X-User-Id` / `X-User-Role` /
   `X-Username` headers (and `?user_id=` query args), so anyone reaching
   port 8081 could impersonate any user, including admins. The proxy now
   stamps an HMAC-signed `X-Internal-Auth` header; ws_server verifies it
   before honoring identity.
   Set `INTERNAL_AUTH_SECRET` (same value) on both processes via
   `config.env`. A 64-hex-char value has been seeded into your local
   `config.env` already.

3. **All `/api/admin/*` endpoints in ws_server now require admin role.**
   `handle_admin_status`, `handle_admin_token`, `handle_admin_fyers_token`,
   `handle_admin_test_broker`, `handle_admin_settings_get`,
   `handle_admin_settings_update`, `handle_admin_trades_export`,
   `handle_admin_trades_clear`, `handle_admin_logs`,
   `handle_admin_rollover`, `handle_admin_settings_update_chat_key` —
   all gated.

4. **`is_admin_path` now matches as a prefix and always covers
   `/api/admin/`.** Previously only the exact `/admin` and `/divergence`
   were treated as admin, so `/api/admin/token` proxied through with no
   role check.

5. **`X-Forwarded-For` is no longer trusted blindly.** The proxy now
   honors it only when the immediate peer is loopback or in a configured
   `TRUSTED_PROXIES` list. Closes the brute-force / IP-spoofing bypass.

6. **OTP generation uses `secrets.randbelow` (CSPRNG)** instead of
   `random.randint`. A new `email_otp_attempts` column caps wrong tries
   per OTP at 5 (then the OTP self-destructs). Rate limits added on
   `/api/auth/verify-email` (10/10m) and `/api/auth/resend-otp` (3/10m).

7. **`SECURE_COOKIES=1` flag** adds the `Secure` attribute to session and
   CSRF cookies. Flip on once HTTPS is in front.

8. **CSP no longer allows `unsafe-eval`.** Verified no `eval()` or
   `new Function()` calls in the served HTML.

9. **Failed logins from unknown usernames no longer count toward the
   IP-level brute-force lockout.** Real failures still do.

10. **`paper_trade_update` PnL formula corrected.** The legacy code
    branched on `direction == "SELL"`, but stored values are
    `BULLISH/BEARISH/CE/PE` — bearish wins were sign-flipped. Both code
    paths (`auth_proxy` and `ws_server`) now use the same option-buyer
    helper in `auth_helpers.compute_option_buyer_pnl`.

## Functional bugs fixed

- **`DB.close()`** referenced a non-existent `self._conn`; now uses the
  thread-local connection and is idempotent.
- **Concurrent-session limit** kicked every other device on the 6th
  login. Now trims only the oldest sessions via
  `delete_oldest_user_sessions(keep=N-1)`.
- **`validate_session` write amplification**: `last_seen` updates are
  throttled to once per minute via the new `touch_session()`.
- **Daily trade limit** for manual entries no longer counts auto-trades.
- **Auto-scan timer's `'bridge' not in dir()`** trick is replaced with
  proper variable scoping; `bridge` is initialized to `None` and reused
  cleanly across the NIFTY-direction and candidate-fetch blocks.
- **CSRF debug log** read the wrong cookie name (`csrf_token` instead of
  `quantra_csrf`); every diag line said `session: None`.
- **WebSocket proxy** now passes identity via the signed header instead
  of editable query-string args.

## New modules

- `internal_auth.py` — HMAC-signed inter-process identity header.
- `auth_helpers.py` — `require_session` / `require_admin` decorator
  factories and a single `compute_option_buyer_pnl` helper.

## Schema migration

- v6 → v7 migration adds `email_otp_attempts INTEGER NOT NULL DEFAULT 0`
  to `users`. Runs automatically on first start.

## What I deliberately did NOT change

These are flagged in the original audit but are large, risky refactors
that need their own session and a real test suite first:

- Splitting the 6,287-line `DashboardServer` class in `ws_server.py` into
  cohesive services.
- Merging `auto_trader.py` and `auto_paper_trader.py` into one engine.
- Introducing a `PaperTrade` dataclass to replace the three different
  dict shapes (DB row, in-memory, `_paper_trade_to_dict` output).
- Razorpay webhook handler for async subscription events.
- Removing `unsafe-inline` from CSP (requires moving inline scripts out
  of the giant HTML files).
- Switching session cookie to `Secure` by default (waits on HTTPS).
- Repository tidy-up (the 60+ `patch_*.py`, `fix_*.py`, `temp*.py`,
  `_check_*.py`, `*.original_backup`, log/PID/db files at the project
  root). I intentionally did not delete anything to keep the change
  reversible.
- Honoring NSE holidays in the auto-trade window.

## Operational notes

- Existing sessions remain valid; the schema migration only adds a column.
- The proxy will fail-closed when ws_server isn't running with the same
  `INTERNAL_AUTH_SECRET` (admin endpoints will 401/403). Make sure the
  same value is in BOTH processes' environment.
- For the GCP deploy, copy the new `INTERNAL_AUTH_SECRET` line to the
  server's `config.env` and bounce both services.
